重點字與觀念小叮嚀

Big Word Alert - 重點字提示

貫穿此系列所有筆記，重點提示文章內所講述到的所有的知識點。
在此，我會用一樣的標籤與字體大小著重標記，並且寫下註解。

Syntax Parsers - 語法解析器

一個程式，如果你的語法是有效的，它能夠讀取程式碼並且決定如何動作。

你寫下了一段程式碼，然後有其他人寫了一個程式，把你寫的程式碼轉換成電腦能看懂的東西，這些程式稱之為「編譯器」。而這些程式，會一字一句讀取你的程式碼，並決定這些語法是否有效，然後將這些程式碼轉換成為電腦能夠讀懂的語言

Lexical Environments - 詞彙環境

這代表程式碼在程式中實際所在的位置。

「詞彙」這個字代表和文法有關，所以這裡在討論的是你寫的程式碼中，它的語法與它的單字。
現在，詞彙環境存在於一些程式語言中，這些程式語言認為程式碼寫在哪很重要，但並不是每個程式語言都是如此，例如：有一個函數，裡面有個變數，這個變數詞彙上在函數裡面，也就是實際上程式碼的位置。

但是要記得一件事情，我們寫的程式碼並不會直接讓電腦執行，而是會透過「轉譯器」轉換成電腦能夠讀懂的語句，但是那些詞彙在程式語言中，所代表的是你所「看到的東西」，它可以給你一些方向，讓你能夠想像到它在電腦記憶體中實際上的位置，也會演示出它會如何與其他變數、函數…等等相互作用。

Execution Contexts - 執行環境

一個環境，幫助管理正在執行的程式。

我們有許多的詞彙環境，也就是程式碼實際上所在的位置，但那個才是現在正在執行的就是被執行環境所管理，執行環境包含了你所寫的程式碼、正在執行的程式碼，但它所包含的不只是你所寫的程式碼。

記得，你的程式碼正在被「轉譯」、正在被另一個程式編輯，所以它正在執行你所撰寫的程式。

Name/Value pairs and Object - 名稱 / 值配對與物件

在 JavaScript 中，「物件的」意義非常重要，我們會花很多時間在這上面，但是在了解物件的意義之前，我們必須先知道「名稱 / 值配對」的意思。

一個「名稱 / 值配對」代表一個名稱會對應到唯一的一個值，名稱可以被定義很多次，但是給定名稱值的情況下，它只會有一個。也就是說，一段正在執行的程式碼，同樣名稱只會有一個，而這個名稱只能被一個值所定義，然而，這個值可能是更多「名稱/值配對」。

而物件，就是「名稱/值配對」的組合，在 JavaScript 中這是物件最簡單的定義，其他程式語言中，物件可能有更多更複雜的概念，但是在 JavaScript 中，物件就是如此而已。

舉例來說：我有一個名稱，名稱有一個值，但這個值可能是兩個「名稱/值配對」，在這些名稱哩，它的值可能又是更多「名稱/值配對」。
